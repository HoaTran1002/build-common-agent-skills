# PR Review

## Purpose
Review code changes for correctness, maintainability, security, and test coverage.

## Inputs
- A git diff, PR description, or list of changed files
- Optional product context or issue link

## Workflow
1. Identify changed files.
2. Group changes by feature or risk area.
3. Review correctness first.
4. Review security and data handling.
5. Review test coverage.
6. Return findings by severity.

## Output format
### Critical
### Major
### Minor
### Suggested tests
### Files reviewed
